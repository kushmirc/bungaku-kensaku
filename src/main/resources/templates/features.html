🏗️ システムアーキテクチャ

ハイブリッド Java + Python アーキテクチャ

Frontend (Thymeleaf) ↔ Spring Boot (Java)
↓
PostgreSQL Database ← → Python AI Service (FastAPI)
↓                                                    ↓
Pinecone Vector DB ← → OpenAI API

Deployment Target: AWS EC2

本システムは、Java Spring Bootを核とした安定性と、Python AIサービスによる高度な言語処理能力を組み合わせたハイブリッドアーキテクチャを採用しています。

🤖 AI機能

セマンティック検索

OpenAI の text-embedding-3-small モデルを使用して、テキストの意味的理解に基づく高度な検索を実現しています。単純なキーワードマッチングではなく、概念や文脈を理解した検索が可能です。

インテリジェント要約

GPT-4を活用して、各検索結果に対して以下の情報を自動生成：

  • 文脈要約：該当箇所の背景と状況説明
  • 関連性説明：なぜその箇所が検索クエリに関連するかの解説
  • 背景情報：より深い理解のための追加コンテキスト

📊 システム性能

4879 インデックス済みチャンク
60 処理済み書籍
1536次元 ベクトル空間

スケーラビリティ

現在のアーキテクチャは60冊から数百冊への拡張を想定して設計されています。モジュラー設計により、各コンポーネントは独立してスケールできます。

🔄 データ処理パイプライン

文書処理フロー

  1. テキスト抽出：デジタル書籍からの高精度テキスト抽出
  2. チャンキング：500トークン単位での文書分割（50トークンオーバーラップ）
  3. エンベディング生成：Python AIサービス経由でOpenAI APIを呼び出し
  4. ベクトル格納：Pineconeへの高速インデックス化
  5. メタデータ保存：PostgreSQLでの構造化データ管理

検索処理フロー

  1. クエリ処理：ユーザー入力の前処理と最適化
  2. ベクトル化：検索クエリの埋め込みベクトル生成
  3. 類似度検索：Pineconeでの高速近似最近傍探索
  4. AI要約生成：GPT-4による文脈理解と要約
  5. 結果表示：ユーザーフレンドリーな形式での結果提示

🛡️ プライバシー保護

メンバーの皆様が安心してご利用いただけるよう、個人情報保護を最優先に設計されています。

匿名アクセス

  • ユーザーアカウント不要の完全匿名検索
  • 検索履歴の保存なし
  • 個人を特定できる情報の収集なし
  • 各検索セッションの完全独立性

データ保護

  • 完全暗号化通信 (HTTPS/TLS)
  • APIキーの安全な管理
  • システム内でのクエリ履歴非保存
  • 個人識別情報の収集なし

メンバー保護の理念

従来の対面での指導相談とは異なり、本システムでは完全な匿名性を保ちながら池田先生の指導にアクセスできます。個人的な悩みや状況を安心して検索していただけ、判断や開示を恐れることなく、プライベートな環境で深い学びを得ることができます。
